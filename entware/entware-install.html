// Руководство по установке

Entware
на Keenetic и Netcraze

Пошаговая инструкция по установке пакетного менеджера entware на внутренний или внешний накопитель роутера

00
Подключение к CLI роутера

Все команды выполняются в командной строке (CLI) роутера. Подключиться можно двумя способами:

🔌 Telnet / SSH: IP 192.168.1.1, порт 22. Логин и пароль — те же, что от веб-интерфейса.
🌐 Встроенная консоль (Web CLI): откройте в браузере
http://192.168.1.1/a и авторизуйтесь.
⚠ Компонент SSH: для подключения по SSH/Telnet в веб-интерфейсе должен быть установлен компонент «Сервер SSH» («Настройки системы» → «Изменить набор компонентов»).
http://192.168.1.1/system/osAndUpdates/components
⏳ Важно: выполняйте команды с небольшим ожиданием. При ошибках — смотрите логи роутера:
http://192.168.1.1/diagnostics/general/log
Подготовка
01
Определить архитектуру роутера

Выполните команду для определения архитектуры процессора:

keenetic cli
show version | grep arch

Пример вывода:

arch: mipsel

Соответствие архитектур и установщиков:

АрхитектураУстановщик
aarch64aarch64-k3.10 / aarch64-installer.tar.gz
mipselmipselsf-k3.4 / mipsel-installer.tar.gz
mipsmipssf-k3.4 / mips-installer.tar.gz
02
Определить имя диска для монтирования

Подключитесь к роутеру через CLI и выполните команду:

keenetic cli
show cifs | grep mount

Пример вывода команды:

automount: yes

share:
  mount: 57c0a37c-ea82-f6bd-27e5-2b495ca59f16:

share:
  mount: 003eae0a-3700-ba4b-b1b7-daa70f89cd16:

share:
  mount: 6dd1526a-d360-4884-a656-ff647036837b:
ℹ Скопируйте нужный UUID из строки mount: — без двоеточия на конце. Этот UUID вводится в шаге 4. Внутренний накопитель — storage:/
Установка
!
Сломанный Entware на внутреннем накопителе — сброс
▼
⚠ Только для внутреннего накопителя. Если Entware сломался — выполните шаги ниже по порядку, затем переходите к установке.
1Отключиться от Entware и сохранить
keenetic cli
no opkg disk
system configuration save
2Отмонтировать накопитель
keenetic cli
no system mount storage:
3Очистить накопитель
keenetic cli
erase storage:
4Примонтировать накопитель
keenetic cli
system mount storage:
→ После выполнения переходите к шагу 3 — установка на внутренний накопитель
03
Установка на внутренний накопитель

Выберите команду в соответствии с архитектурой роутера и выполните её в CLI:

▸ aarch64

aarch64
opkg disk storage:/ https://bin.entware.net/aarch64-k3.10/installer/aarch64-installer.tar.gz

▸ mipsel

mipsel
opkg disk storage:/ https://bin.entware.net/mipselsf-k3.4/installer/mipsel-installer.tar.gz

▸ mips

mips
opkg disk storage:/ https://bin.entware.net/mipssf-k3.4/installer/mips-installer.tar.gz
04
Установка на внешний накопитель

Введите UUID из строки mount: (шаг 2) без двоеточия на конце:

MOUNT-POINT
✓Команды готовы — выберите архитектуру и скопируйте

▸ aarch64

aarch64 — внешний

▸ mipsel

mipsel — внешний

▸ mips

mips — внешний
ℹ Вставьте UUID из строки mount: выше (без двоеточия) — команды появятся мгновенно
05
После установки

После успешной установки обновите список пакетов:

keenetic cli
opkg update && opkg upgrade
✓ Теперь можно устанавливать пакеты: opkg install <имя_пакета>. Например: opkg install wget-ssl curl && opkg remove wget-nossl
